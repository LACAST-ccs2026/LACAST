# RQ3: Contribution Evaluation of Components

This repository contains the analysis code for Research Question 3 (RQ3) evaluating the contribution of key components in LACAST.

## Quick Start

### Requirements

- Python 3.7+
- Required packages: matplotlib, numpy, scipy

```bash
pip install matplotlib numpy scipy
```

### Run Analysis

Execute the complete analysis pipeline:

```bash
python run_analysis.py
```

This will automatically:
1. Process SQL queries from all LACAST configurations
2. Calculate correctness and token diversity metrics
3. Generate comparison tables

### Output

- **Performance Metrics**: `output/metrics_final.csv`
  - Correctness (average %)
  - Unique tokens (average)

## Input Dataset

The input data consists of SQL queries generated by three LACAST configurations:

- **input_dataset/LACAST/**: Full LACAST system
- **input_dataset/LACAST_I/**: Without Iterative Training Agent
- **input_dataset/LACAST_S/**: Without Static Constraints

Each database directory contains `epoch-1.jsonl` files with the following format:
```json
{"sql": "SELECT * FROM table1", "result": "success", "error_info": ""}
```

### Supported Databases

- MySQL
- MariaDB
- PostgreSQL
- SQLite
- TiDB

## Configuration Descriptions

- **LACAST (Full)**: Complete system with both Static Constraints and Iterative Training Agent
- **LACAST-I**: Without Iterative Training Agent, only Constraint-Aware Drafting Agent
- **LACAST-S**: Without Static Constraints, uses unconstrained AST-based generation
