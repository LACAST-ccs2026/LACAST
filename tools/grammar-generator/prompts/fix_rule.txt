=== CORE CONCEPTS (Evaluators) ===
Evaluators are non-terminal symbols that represent expression result categories.

There are exactly TWO evaluator types:
1. `<datatype>_evaluator_without_agg` — for non-aggregate expressions
2. `<datatype>_AGG_evaluator` — for aggregate expressions

Each rule connects an evaluator to its definition:
  <type_eval> : <left>   — the rule belongs to this evaluator
  <left> : <right>       — the rule definition

=== CORRECT RULE EXAMPLES (Study these — JSON format) ===

Non-aggregate function, no agg params:
  {{"type_eval": "integer_evaluator_without_agg", "left": "ABS_integer_evaluator",
   "right": "\"ABS(\" _var_integer \")\""}}

Non-aggregate function, with agg params (left gets agg_ prefix):
  {{"type_eval": "integer_evaluator_without_agg", "left": "agg_ABS_integer_evaluator",
   "right": "\"ABS(\" integer_AGG_evaluator "\""}}

Aggregate function:
  {{"type_eval": "integer_AGG_evaluator", "left": "CNT_integer_evaluator",
   "right": "\"COUNT(\" _var_integer \")\""}}

With const_productions (string literals):

   "right": "\"DATE(\" _var_text \",\" date_modifier \")\"",

With const_productions (numeric literals — must be quoted in grammar):

IMPORTANT about JSON escaping:
  In these examples, `\"` in the JSON string produces a raw `"` character.
  Do NOT write `\\\"` — that would produce a literal backslash before the quote.
  After JSON.parse: "\"ABS(\" _var_integer \")\"" → "ABS(" _var_integer ")"
